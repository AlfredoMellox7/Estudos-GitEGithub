git clone ( o url do repositório ), --branch ( nome da branch ) --single-branch //Puxa somente uma branch.

***Ao fazer alguma alteração em um clone, dar o add, commit e depois o git push origin main .

git log = vê as configurações do commit, pra qual branch, o autor, data e hora .

git ignore = é para quando você não quiser adicionar tal pasta no commit . O git não reconhecerá .

git keep = é para um repositório vazio mas com um plano para ele .

git branch (nome da nova branch)= irá criar uma nova branch.

git checkout (nome da nova branch) = irá mudar para a branch com o nome desejado .

Voltar para a branch main e dar merge com a branch que você criou( ex : estando na main = git merge versao_01 ).

git branch = mostra os nomes das branchs.

PR = Pull request .

Ao dar fork em um repositório de outra pessoa, você irá clona-lo para seu GitHub assim para fazer mudanças, e até clona-lo para seu repositório local .
Assim com as mudanças feitas podendo dar um pull request para os dono dar merge com o original dele.

git tag -a (nome de tag ex : v1.1 ) e depois dar git push ex : git push origin v1.1
( tag serve pra fazer um lançamento do projeto ex = Tarkov beta versão 31.7 . A tag 31.7 vai ter aquela versão.)
